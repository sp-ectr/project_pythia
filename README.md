# project_pythia

Цифровой таро-сервис, работающий внутри Telegram Mini App.
Делался ради обучения, но в процессе перерос в нечто большее.

Собран на актуальных коммерческих веб/бек технологиях и практиках.

## Architecture

Асинхронный stateless бэкенд + детерминированный фронт на React, работающий как конечный автомат.

## Key Features

- Голосовой и текстовый ввод запроса
- ИИ-премодерация промптов перед раскладом
- Атомарное списание токенов на уровне БД, без гонок при параллельных запросах
- Фоновая отправка готового расклада юзеру в Telegram

## Principles

- Никакой магии — прозрачная логика вместо тяжёлых абстракций
- БД — источник правды для бизнес-логики
- Детерминированный UI без гонки состояний
- Минимум зависимостей, максимум производительности

## Tech Stack

**Backend:** Python 3.11+, FastAPI, SQLAlchemy 2.0 (Async), PostgreSQL

**Frontend:** React 19, TypeScript, Vite, Tailwind CSS v4

**AI/ML:** Gemini API, Groq Whisper

**Infra/DevOps:** Docker, GitHub Actions (CI/CD)
